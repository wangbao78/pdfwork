# PDF 工具箱 — 需求文档

## 1. 核心用户问题

用户在处理 PDF 文件时，遇到的不是"缺软件"，而是"太麻烦"——需要打开臃肿的桌面软件、注册账号、被广告打断，最终可能还要付费才能导出。三个最高频痛点：

- **格式转换的门槛**：收到一份 PDF 合同/报告，需要编辑修改，但没有 Word 版，也不想装几百 MB 的 Acrobat
- **文件合并的繁琐**：HR 收集了 10 份 PDF 简历、财务汇总了多个发票扫描件，手动合并要么装专业软件，要么截图拼凑
- **文件太大发不出去**：PDF 导出时未压缩，超过了邮件附件上限（25MB）或招聘系统限制，需要降体积但不丢清晰度

## 2. 典型用户故事

> **故事 1 — 求职者改简历**
> 作为一个正在找工作的人，我希望把 PDF 简历转成 Word 格式，以便于修改内容后再次导出为不同版本的简历投递不同公司。

> **故事 2 — 行政汇总报销单**
> 作为一个行政人员，我希望把多个同事发来的 PDF 发票合并成一个文件，以便于一次性提交给财务系统，不用打印扫描。

> **故事 3 — 卖家压缩产品手册**
> 作为一个在电商平台开店的卖家，我希望压缩产品手册 PDF 到 5MB 以下，以便于通过平台的附件限制上传到商品详情页。

> **故事 4 — 学生合并论文资料**
> 作为一个研究生，我希望把从知网下载的多篇 PDF 参考文献合并成一个文件，以便于整理文献综述时集中阅读和标注。

## 3. 目标用户类型

| 用户类型 | 特征 | 高频场景 | 付费意愿 |
|----------|------|----------|----------|
| **白领上班族** | 每天和文档打交道，非技术背景 | 转 Word 编辑、合并报表、压缩后发邮件 | 中（公司可报销） |
| **自由职业/小商家** | 一人多角色，需要快速出文件 | 合并合同附件、压缩产品图册 | 中低（自己掏钱） |
| **学生/研究人员** | 低频但集中使用（论文季） | 合并文献、格式互转 | 低（有免费够用） |

## 4. 产品价值（一句话）

**让任何人无需安装任何软件，在浏览器里三步完成 PDF 的转换、合并和压缩，省掉 Acrobat 的臃肿和收费。**

---

## 5. 功能目标

解决 PDF 三个最高频操作的"最后一公里"：**转换、合并、压缩**。用户打开网页 → 拖文件 → 点按钮 → 下载结果，全程不超过 30 秒。

不追求功能数量，追求每个功能"做完即走"的流畅度。

## 6. 核心功能说明

### 6.1 PDF 转 Word

| 项目 | 说明 |
|------|------|
| 功能描述 | 上传 PDF，后台转为可编辑的 .docx 文件 |
| 输入限制 | 文件 ≤100MB（Pro）/ ≤5MB（Free），仅限 .pdf |
| 输出格式 | .docx |
| 处理方式 | LibreOffice headless 服务端转换 |
| 转换质量预期 | 文字内容完整保留，复杂排版（表格、分栏）可能有偏差 |

### 6.2 合并 PDF

| 项目 | 说明 |
|------|------|
| 功能描述 | 上传多个 PDF，按指定顺序拼接为一个文件 |
| 输入限制 | 单个文件 ≤100MB（Pro）/ ≤5MB（Free），最多 20 个文件 |
| 输出格式 | .pdf |
| 处理方式 | pdf-lib 服务端合并 |
| 交互要点 | 上传后可拖拽调整文件顺序 |

### 6.3 压缩 PDF

| 项目 | 说明 |
|------|------|
| 功能描述 | 上传 PDF，按选择的压缩级别减小文件体积 |
| 输入限制 | 文件 ≤100MB（Pro）/ ≤5MB（Free） |
| 输出格式 | .pdf |
| 压缩级别 | 标准（推荐）、高压缩（文件更小画质下降）、极限（Pro 专属） |
| 处理方式 | Ghostscript 服务端压缩 |
| 交互要点 | 压缩后显示前后文件大小对比 |

## 7. 用户主流程

三个工具共用同一套核心流程：

```
进入首页 → 选择工具（转Word / 合并 / 压缩）
  → 上传文件（拖拽或点击）
  → 调节参数（仅压缩需选级别）
  → 点击"开始处理"
  → 等待处理（显示进度动画）
  → 自动下载结果
  → 返回首页继续下一个任务
```

合并 PDF 多一步：上传后 → **拖拽调整文件顺序** → 再开始合并。

## 10. 非目标范围（本期不做）

明确砍掉，避免战线拉长：

- PDF 拆分（一个文件拆成多页）
- PDF 旋转/提取页面
- PDF 加密/解密
- PDF 转 Excel/PPT/图片
- 批量文件夹上传
- 在线预览编辑
- OCR 文字识别
- API 接口对外开放
- 团队协作功能

这些可以后续根据用户反馈再判断，先集中精力把三个核心工具做到打磨光滑。

## 11. 边界条件

### 11.1 内容侧边界

| 边界场景 | 处理方式 | 是否写入 PRD |
|----------|----------|:--:|
| 用户上传的不是 PDF（.jpg/.png/.docx 伪装后缀） | 前端检查 MIME 类型 + 服务端检查文件头 Magic Number，不匹配则拒绝并提示"请上传 PDF 文件" | ✅ 是 |
| PDF 被密码保护，无法读取内容 | LibreOffice 会报错，捕获异常后提示"该 PDF 已加密，请先解除密码保护后重试" | ✅ 是 |
| PDF 是扫描件（图片型 PDF），没有可选文字 | PDF→Word 转换后会是空白 Word，需要检查输出文件大小和文本长度，若接近零则提示"该 PDF 为扫描件，文字识别（OCR）功能请升级 Pro" | ✅ 是 |
| PDF 内嵌字体未安装（特殊字体被替换） | 转换后排版可能偏移。处理时不阻断，但在下载页标注"因字体兼容性，排版可能有细微偏差" | ❌ 不写，作为已知限制备查 |
| PDF 页数过多（500+ 页）导致处理超时 | 设置页数上限 200 页（Free 50 页），超限提示"PDF 页数超过限制，请拆分后重试" | ✅ 是 |
| 合并时多个 PDF 页面尺寸不一致（A4 + Letter + 自定义） | 输出统一为 A4，小页面居中保留，不裁剪。提示"页面尺寸已统一为 A4" | ✅ 是 |
| 用户上传含病毒的伪装文件（.exe 改 .pdf） | 服务端校验文件头 Magic Number（%PDF），不合规直接拒绝，文件不落盘超过 1 秒 | ✅ 是 |

### 11.2 用户侧边界

| 边界场景 | 处理方式 | 是否写入 PRD |
|----------|----------|:--:|
| 免费用户今天已用完 3 次，再次点击"开始处理" | 按钮置灰，弹窗提示"今日免费额度已用完，明天再来或升级 Pro 无限使用"，弹窗内置"查看 Pro"按钮跳转定价页 | ✅ 是 |
| 免费用户上传超过 5MB 的文件 | 上传阶段即拦截，提示"免费版单文件最大 5MB，Pro 支持 100MB"，附带升级入口 | ✅ 是 |
| Pro 用户订阅到期未续费 | 到期当天降级为 Free 配额，正在处理中的任务不受影响，处理完仍可下载。下次使用时按 Free 规则校验 | ✅ 是 |
| 未登录用户直接访问工具页 URL（如 `/tools/pdf-to-word`） | 中间件拦截，跳转登录页，登录成功后自动回到原工具页 | ✅ 是 |
| 用户短时间内批量调用（脚本刷量） | 单用户每分钟最多处理 5 个任务，超频返回 429 + "操作太频繁，请稍后再试"。Pro 放宽到 20 个/分钟 | ✅ 是 |
| 用户上传含个人隐私的文件（合同、简历、身份证） | 处理完成后文件立即从 R2 删除，1 小时生命周期兜底。首页明确标注"处理完即删除，不存储你的文件"。不写进 PRD，但必须有此机制 | ❌ 不写 |

### 11.3 状态侧边界

| 边界场景 | 处理方式 | 是否写入 PRD |
|----------|----------|:--:|
| 文件上传中用户关闭了页面 | 上传到 R2 一半的文件因不完整，1 小时后自动过期。前端监听 `beforeunload` 弹窗挽留："文件正在上传，确定离开？" | ✅ 是 |
| 处理过程中服务器异常（LibreOffice 崩溃、内存不足） | 后端捕获异常，数据库记录 ERROR 状态。前端轮询到失败后显示"处理失败，请重试"，不扣当日额度。失败超过 3 次提示"服务繁忙，稍后再试" | ✅ 是 |
| 用户上传后长时间不点"开始处理" | 无操作 30 分钟后 R2 文件过期，前端点击按钮时服务端检查文件是否存在，不存在则提示"文件已过期，请重新上传" | ✅ 是 |
| 合并时某个子文件处理失败，但其他已完成 | 采用"全部成功 or 全部回滚"策略，任何一个子文件失败整个任务标记失败，不生成半成品 | ❌ 不写，技术实现细节 |
| 用户在处理完成前点击浏览器后退 | 前端路由守卫拦截，弹窗"处理正在进行，离开将丢失结果"，用户确认后中止 | ❌ 不写（体验优化项，后续版本加） |
| 并发任务过多导致队列拥塞 | 后端实现任务队列（BullMQ），超出处理能力的任务排队等待。队列深度超过 100 时前端显示预估等待时间 | ✅ 是 |

### 11.4 异常侧边界

| 边界场景 | 处理方式 | 是否写入 PRD |
|----------|----------|:--:|
| 服务端 LibreOffice 转换超时（单文件超过 60 秒） | 设置 60 秒超时，超时返回"文件较大，处理超时。Pro 支持更大文件优先处理"，引导升级 | ✅ 是 |
| Ghostscript 压缩后体积不减反增（已经高度压缩的 PDF） | 返回原文件并提示"该文件已被优化，无法进一步压缩"，不扣额度 | ✅ 是 |
| R2 存储服务不可用（Cloudflare 宕机） | API 返回 503，前端显示"服务暂时不可用，请稍后重试"。首页增加状态指示器（可选） | ✅ 是 |
| 用户网络断开，下载到一半失败 | 下载页保留"重新下载"按钮，结果文件 1 小时内有效 | ✅ 是 |
| 数据库连接失败，无法写入用量记录 | 先允许处理（允许短暂写库失败），异步重试写入。超过 5 分钟未恢复则返回 503 | ❌ 不写（运维层问题） |
| Docker 容器重启导致正在处理的任务丢失 | 任务持久化到数据库（PENDING 状态），容器重启后扫描 PENDING 任务重新执行 | ❌ 不写（技术实现细节） |

### 11.5 权限/风控侧边界

| 边界场景 | 处理方式 | 是否写入 PRD |
|----------|----------|:--:|
| 同一 IP 批量注册多个免费账号刷额度 | 注册时校验 IP 频率，单 IP 每小时最多注册 3 个账号。超出后该 IP 被临时禁止注册 24 小时 | ✅ 是 |
| 免费用户通过清除 Cookie/换浏览器绕过每日 3 次限制 | 用量存储在服务端数据库，按 userId + 日期校验，清除前端状态无效 | ❌ 不写（技术实现自明） |
| 用户上传违法内容（如盗版、色情、涉密文件） | 不主动扫描内容（避免隐私争议），但在使用条款中写明"不得用于违法用途"。若收到举报，封禁账号并保留证据 | ✅ 是（写入用户协议） |
| 恶意用户上传超大文件耗尽服务器磁盘 | 上传到 R2 而非本地磁盘，处理时流式读取，不完整文件不过夜 | ❌ 不写（架构设计已覆盖） |
| Stripe 订阅状态与服务端不同步（用户付了钱但权限没更新） | Stripe Webhook 处理 `checkout.session.completed` 即时更新。增加定时任务每小时全量对账一次，发现漏掉的补升 | ✅ 是 |
| 用户申请退款/争议（Chargeback） | Stripe 后台处理退款后 Webhook 触发降级，用户下次访问时自动变为 Free 配额 | ✅ 是（写入 Stripe 配置文档） |
| 调试日志泄露用户文件名 | 日志中不记录用户原始文件名和文件内容，使用任务 ID 作为唯一标识 | ❌ 不写（安全规范自明） |

| I2 | 点击上传 | 点击上传区 | 弹出系统文件选择器，仅显示 .pdf 文件（或默认筛选） |
| I3 | 合并文件排序 | 合并工具中上传 4 个 PDF | 每个文件显示拖拽手柄，拖动第 4 个到第 1 位，顺序更新且不丢文件 |
| I4 | 合并文件删除 | 合并工具中删除其中一个文件 | 该文件从列表移除，其余文件保持原顺序 |

### 12.5 数据埋点验收

| # | 验收项 | 事件名 | 触发时机 | 上报字段 |
|---|--------|--------|----------|----------|
| M1 | 首页访问 | `page_view_home` | 用户进入首页 | page, referrer, timestamp |
| M2 | 工具页进入 | `tool_enter` | 用户进入任一工具页 | tool_name(convert/merge/compress), plan(Free/Pro), timestamp |
| M3 | 文件上传成功 | `file_uploaded` | 文件校验通过并开始上传 | tool_name, file_size, file_pages, plan, timestamp |
| M4 | 处理提交 | `process_start` | 用户点击"开始处理" | tool_name, file_size, plan, is_batch(合并专用), compress_level(压缩专用), timestamp |
| M5 | 处理完成 | `process_done` | 服务端返回结果，触发下载 | tool_name, duration_ms, result_size, plan, success(boolean), timestamp |
| M6 | 处理失败 | `process_fail` | 服务端返回错误 | tool_name, error_type(timeout/crash/oom/encrypted), plan, timestamp |
| M7 | 额度耗尽提示展现 | `quota_exhausted_shown` | 免费用户第 4 次点击处理时弹窗出现 | plan, daily_usage_count, timestamp |
| M8 | 升级入口点击 | `upgrade_click` | 用户点击任何"升级 Pro"按钮/链接 | source(quota_modal/file_size_reject/tool_page/pricing_page), plan, timestamp |
| M9 | 定价页访问 | `page_view_pricing` | 用户进入定价页 | plan, referrer, timestamp |
| M10 | 注册完成 | `signup_complete` | 用户完成注册 | provider(google/email), timestamp |
| M11 | 订阅完成 | `subscription_complete` | Stripe Webhook 确认支付成功 | plan(Pro), amount(9.9), currency(USD), timestamp |
| M12 | 用户留存（次日） | `user_return` | 用户次日再次访问 | days_since_signup, plan, timestamp |

**埋点说明**：
- M1-M6 是核心漏斗数据，能算出每一步的转化率和流失点
- M7-M9 是付费转化数据，能算出"额度告急 → 浏览定价 → 下单"的转化
- M10-M12 是增长数据，看注册率和留存
- 所有埋点不含用户个人信息和文件内容，仅含工具使用指标
