# 管理端方案

## 概述

可视化后台看板，仅管理员可访问。展示用户数据、使用统计。

## 权限方案

**环境变量 `ADMIN_EMAIL`**：在 Railway 设置一个管理员邮箱（你的邮箱 `wangbao8866@sohu.com`）。访问 `/admin/*` 时检查当前登录用户的 email 是否匹配，不匹配返回 403。

- 不需要改数据库 schema
- 不需要角色字段
- 你换邮箱只需改环境变量

流程：
```
访问 /admin → 未登录 → 跳登录页
          → 已登录但 email ≠ ADMIN_EMAIL → 403 "无权访问"
          → 已登录且 email = ADMIN_EMAIL → 显示管理端
```

## 数据来源

全部从 Prisma 查，已有字段足够：

| 指标 | 查询 |
|------|------|
| 总用户数 | `db.user.count()` |
| 今日注册 | `db.user.count({ where: { createdAt: { gte: todayStart } } })` |
| Free/Pro 人数 | `db.user.count({ where: { plan } })` |
| 总文件数 | `db.file.count()` |
| 今日处理 | `db.user.aggregate({ sum: { dailyUsage } })` 或查 File 今日数量 |
| 用户列表 | `db.user.findMany({ orderBy: { createdAt: "desc" }, take, skip })` |
| 游客使用 | `db.guestUsage.aggregate({ sum: { count }, where: { date: today } })` |
