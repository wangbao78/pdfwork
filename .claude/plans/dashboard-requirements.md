# 仪表盘需求文档

## 概述

登录后的个人中心页面，展示用户套餐信息和使用统计。当前套餐卡片正常，使用统计卡片为硬编码占位数据，需接入数据库真实数据。

## 数据来源

`User` 表（Prisma）已有字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `plan` | `FREE \| PRO` | 套餐类型 |
| `dailyUsage` | `Int` | 当日使用次数 |
| `lastUsageDate` | `DateTime?` | 最后使用日期（用于跨天归零） |
| `totalUsage` | `Int` | 累计使用次数 |

限额配置（`src/lib/access.ts`）：Free 5次/天、10MB、30页；Pro 不限次、100MB、200页。

### 状态变体

**首次使用（totalUsage = 0）**

```
│  今日已用                        │
│  0/5 次                         │
│  ░░░░░░░░░░ 0%                  │
│                                 │
│  累计处理：0 次                  │
│  欢迎使用！试试 PDF 转 Word →     │
```

**跨天（lastUsageDate 不是今天）**
- dailyUsage 归零显示 `0/5`，系统在 `trackUsage` 时自动重置，仪表盘只读不做重置逻辑

**用满（dailyUsage ≥ 5）**

```
│  今日已用                        │
│  5/5 次  已达上限                │
│  ██████████ 100%（红色进度条）    │
│                                 │
│  [升级 Pro 不限次数 →]           │
```

**数据库不可用**
- 文案回退为 `--/5 次`，进度条不渲染，不报错不白屏


### 使用统计卡片

```
┌─ 使用统计 ──────────────────────┐
│                                 │
│  今日已处理                      │
│  12 次        ·不限次·           │
│                                 │
│  累计处理：1,247 次              │
│  单文件上限：100 MB              │
│  单文件页数：200 页              │
└────────────────────────────────┘
```

- 不放进度条，不显限制数字
- "不限次"用绿色小标签
- 数据库不可用同样回退为 `--`

**关于"本月处理"**：当前数据库没有按月存，只有 `totalUsage`（累计）和 `dailyUsage`（当日）。Pro 的"本月"统计属于锦上添花，暂不实现，后续需要时在 User 表加 `monthlyUsage` + `lastMonthDate` 字段。

## 快速工具栏

现有三个快捷入口（PDF转Word、合并PDF、压缩PDF）保持不变，Free 和 Pro 共用。

## 实现要点

1. 页面已是服务端组件 + `auth()` 鉴权，直接在 `DashboardPage` 里 `await db.user.findUnique({ where: { id: user.id } })` 查数据，不需要额外 API
2. `isNewDay` 判断逻辑：`lastUsageDate` 的日期部分 ≠ 今天 → dailyUsage 显示为 0
3. 不修改数据库，只读查询
4. 数据获取用 try/catch 包裹，DB 不可用时所有数值显示 `--`
5. 文件大小/页数上限从 `access.ts` 的常量引用，不硬编码到模板

## 不变更范围

- 套餐类型、限额数值、Pro 工具列表均不改动
- 定价页、导航栏、首页不受影响
- middleware 不在此需求范围内
